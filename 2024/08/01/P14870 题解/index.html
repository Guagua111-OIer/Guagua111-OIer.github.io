P14870 ICPC 2020 Yokohama R 题解

P14870 题解

题目传送门

先叠个甲，本题解与楼下大佬思路与代码相仿，只有细节与文字可能略有不同（可能会更详细一些吧），希望管理员点个过。如果这篇题解没有看懂（或者比较喜欢短小精悍的题解）可以看一下楼下的题解，也可以在评论区提出问题与建议！

思路

先来转化一下题目要求的东西。

题目主要是要通过一个阈值将图分为若干部分，每一个部分（或者说连通块）都只能全体节点都不比阈值大或着都比阈值大。题目将这个图叫做子图。

如果说我们可以通过给一些个点添加恰好一条边将这些连通块重新连起来形成一整块，这个子图就是合法的。求一个最小的可以令子图合法的阈值，没有输出 -1。

首先虽然有“最小”、“可行”的字眼，但是想想便知这题答案没有单调性所以不能二分。

我们可以将子图的连通块分为两个部分（注意一个部分中可能有多个连通块），左部分全体节点不超过阈值，右部分全体节点都超过了阈值。我们令左部分的节点数量为 szszsz，那么右部分节点数量就是 n−szn - szn−sz。同时令 lll 为左部分的连通块数量，rrr 为右部分的连通块数量。我们把每个连通块看成一个点，那么连通这些连通块就相当于连通一共 l+rl + rl+r 个点，根据树的性质可知，至少只要 l+r−1l + r - 1l+r−1 条边便一定可以连通。

我们继续考虑，什么时候这张图不合法呢？就是 l+r−1l + r - 1l+r−1 条边不能在满足连通的情况下继续满足“给一些个点添加恰好一条边”的要求。我们想按照这个要求，最多应该能添加 min⁡(sz,n−sz)\min(sz, n - sz)min(sz,n−sz) 条边。即在两个部分中选一个较为小的，然后给其中每个点都连一条边。此时边数与点数相等，并且添加任何一条连接两个部分的边都会违反要求。

说了这么多，我们终于可以把合法转换为：

l+r−1≤min⁡(sz,n−sz)l + r - 1 \le \min(sz, n - sz)l+r−1≤min(sz,n−sz)

看到这么多连通块，想必大家脑子里的并查集都已经跃跃欲试了。

如果直接枚举每个关联值，值域太大时间复杂度会直接爆掉，所以需要离散化。但是即便离散化，不加其他优化的时间复杂度，由于并查集要清空让每轮枚举都至少有 O(n)\mathcal O(n)O(n)，总复杂度依然 O(n2)\mathcal O(n^2)O(n2)，难以通过本题。

现在我们转换思路，来重点观察一下图中的边，发现等题目删完边之后，剩下的边（这里设边连接的两个节点为 uuu 和 vvv，令阈值为 kkk，lll 如题）一定满足 lu≤k,lv≤kl_u \le k, l_v \le klu​≤k,lv​≤k 或者 lu>k,lv>kl_u > k, l_v > klu​>k,lv​>k，也就是 max⁡(lu,lv)≤k\max(l_u, l_v) \le kmax(lu​,lv​)≤k 或 min⁡(lu,lv)>k\min(l_u, l_v) > kmin(lu​,lv​)>k，由于每条边的最小值与最大值固定，我们可以将这两个值作为比较的筹码。不满足这些条件的边会被删掉。

我们来分开维护左部分的边与右部分的边，每次增加阈值就只需要将不超过阈值的边从右部分移到左部分即可。也就是可以先遍历左部分的边（阈值增加节点增加），再遍历右部分（阈值增加节点减少）。使用变量来维护左右部分的节点数量，并存在 lll 与 rrr 数组里。最后，我们就可以轻松的使用上面推出的公式做出本题了！
